# 当前进度汇报

日期：2026-06-30

## 已完成

### 1. MinerU 知识库解析服务

已新增独立 Python 服务目录：`km-ingest-service/`。

当前服务已完成 Stage 1 能力：

- 接收本地文件或 HTTP 上传文件。
- 将原始文件上传到 MinIO：`raw/{doc_id}/source.<ext>`。
- 调用 MinerU `pipeline` 后端解析 PDF。
- 将解析产物上传到 MinIO：`parsed/{doc_id}/content.md`、`middle.json`、`layout.json`、图片等。
- 返回 manifest，包含原文件路径、Markdown 路径、JSON 路径、图片路径和错误信息。

### 2. 命令行调用方式

CLI 已可用，示例：

```powershell
cd km-ingest-service
.venv\Scripts\python -m app ingest --file D:\path\sample.pdf --doc-id doc_1 --manifest .work\doc_1\manifest.json
```

MinIO-only smoke test 可使用：

```powershell
cd km-ingest-service
.venv\Scripts\python -m app ingest --file .work\sample.txt --doc-id smoke_test --skip-mineru --manifest .work\smoke_test\manifest.json
```

### 3. HTTP 接口

已新增 FastAPI 接口，方便后续 Spring Boot 或其他服务调用。

启动方式：

```powershell
cd km-ingest-service
.venv\Scripts\uvicorn app.api:app --host 0.0.0.0 --port 8093 --reload
```

接口：

- `GET /health`：健康检查，返回 `{"status":"ok"}`。
- `POST /ingest`：multipart 上传接口。

调用示例：

```powershell
curl.exe -F "file=@D:\path\sample.pdf" -F "doc_id=doc_1" -F "skip_mineru=false" http://localhost:8093/ingest
```

`doc_id` 规则：1-128 个字符，只允许字母、数字、点、下划线、短横线，并且必须以字母或数字开头。

### 4. MinIO 配置结论

本机当前 MinIO 端口结论：

- `http://localhost:9000` 是 MinIO Console。
- `localhost:9005` 是 S3 API 端口。

因此 `km-ingest-service/.env` 中应使用：

```env
MINIO_ENDPOINT=localhost:9005
MINIO_BUCKET=km-documents
MINIO_SECURE=false
```

MinIO 账号密码已由本地 `.env` 保存，不应提交到 Git。
