# Markdown 表格生成规范

> 供章节正文 Prompt 引用及后端 DOCX 导出前校验使用。

## 1. 基本格式

```markdown
| 序号 | 检查项目 | 检查结果 | 备注 |
| --- | --- | --- | --- |
| 1 | 主变压器油温 | 正常 | 最高 72℃，低于报警值 |
| 2 | 冷却器运行 | 正常 | 双路电源均可用 |
```

规则：

- 第二行必须是 `| --- | --- |` 形式的分隔行
- 每行列数必须与表头一致
- 单元格内避免换行；长文本用简短表述
- 表格前后各保留一个空行

## 4. Prompt 中的表格指令示例

在 `promptHint` 或 User Prompt 中可写：

- 「需含 4 列设备检查项表格：序号、检查项目、检查结果、备注」
- 「需含库存盘点对比表，列：煤种、账面、实盘、差异、差异率」
- 「需含 3～5 条审计发现问题表格」

## 5. 常见错误（LLM 输出需避免）

| 错误 | 正确做法 |
|------|----------|
| 缺少分隔行 `\| --- \|` | 表头下必须加分隔行 |
| 列数不一致 | 每行列数与表头相同 |
| 使用 HTML `<table>` | 只用 Markdown 表格 |
| 在表格上方写「表 1 xxx」 | 题注由 DOCX 导出模块自动添加 |
| 合并单元格语法 | Markdown 不支持，拆成多行描述 |
