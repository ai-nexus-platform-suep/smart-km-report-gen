## 1. 方案摘要

| 项 | 知识管理组约定 |
|----|----------------|
| 业务主服务 | **Java** `km-backend`（知识库、文档、检索 API、状态机、对外契约） |
| AI 处理服务 | **Python** `km-ai-service`（**chunk 切分、embedding、rerank**） |
| 向量库 | **Milvus**（三组统一选型，本组写入与检索） |
| 关系库 | MySQL 8.x（业务元数据、用户关联见 `docs/user.sql`） |
| 对象存储 | MinIO（原始文件） |
| 消息队列 | RabbitMQ（文档异步处理触发） |
| 缓存 | Redis（任务状态 / 会话等） |
| 部署形态 | **服务并列**（与 `middleware-spec` 微服务思路一致，经 Gateway/Nacos 注册） |

**对外接口不变**：问答组/报告组仍只调用 Java 的 `docs/api-contract.yaml`（如 `POST /api/search`），**不直连 Python**。

## 3. 职责边界（与大组对齐）

### 3.1 Java `km-backend`（其他正常）

- 用户/权限（EPIC-01，用户表见 `docs/user.sql`）
- 知识库 CRUD、文档上传与列表、状态展示
- 文件存 MinIO，元数据存 MySQL
- 发送 RabbitMQ 消息触发处理
- 接收 Python 回调或轮询，更新 `document.status`
- 对外 `api-contract.yaml` 全部 REST（含 `/api/search` 入口）
- 通过 **OpenFeign / HTTP** 调用 `km-ai-service` 的 rerank（检索链路）

### 3.2 Python `km-ai-service`（chunk + embed + rerank）

| 能力 | 说明 |
|------|------|
| 解析 | PDF/DOCX 等 → 纯文本（如 unstructured / pypdf / python-docx，实现阶段选型） |
| **chunk 切分** | 按知识库 `chunk_strategy_json`：标题层级 / 固定长度+overlap |
| **embedding** | 调用 SiliconFlow 或本地模型，输出向量 |
| **rerank** | 对召回候选重排序，返回分数 |
| Milvus | 写入 chunk 向量；按约定提供向量检索或仅写入由 Java 调 Milvus SDK |

内部接口契约：**`docs/ai-service-contract.yaml`**

### 3.3 检索链路（`POST /api/search`）

```
客户端 → Gateway → km-backend
  → Milvus 向量召回（Java 或 Feign 调 Python /search）
  → km-ai-service /internal/rerank（若 searchMode=vector_rerank）
  → km-backend 组装 api-contract 响应
```

> **待大组确认一项**：Milvus 检索由 Java 直连 SDK，还是统一走 Python 封装；本组倾向 **Python 负责写+检索向量，Java 只编排**，减少双端维护 Collection 逻辑。

## 4. 文档处理流水线

```
[Java] 上传 → MinIO + MySQL(UPLOADED) → RabbitMQ
                    │
                    ▼
[Python] 消费任务 → 读 MinIO → 解析 → chunk → embed → 写 Milvus + MySQL chunk 表
                    │
                    ▼
[Java] 状态更新 → READY / FAILED（HTTP 回调或 MQ 回执）
```

状态机（与 PRD 一致）：

`UPLOADED → PARSING → CHUNKING → EMBEDDING → READY | FAILED`

状态字段由 **km-backend** 对外展示；各阶段可由 **km-ai-service** 通过内部 API 回调更新。

## 5. 与大组 `middleware-spec.md` 的对齐

| 大组规范 | 知识管理组落地 |
|----------|----------------|
| Nacos 注册发现 | `km-backend`、`km-ai-service` 均注册；Namespace：`dev` / `test` / `prod` |
| OpenFeign | `km-backend` 内 `client/KmAiClient` 调用 Python 内部 API |
| 内部路径前缀 | Python 使用 `/internal/*`，与对外 `/api/*` 区分 |
| 统一响应 | 建议全组统一 `ApiResponse`；当前 Java 用 `code=0`，联调前与大组确认是否改为 `200` |
| 敏感配置 | API Key、DB 密码放 Nacos / 环境变量，不进 Git |
| Gateway 路由示例 | `/api/km/**` → `km-backend`；`/internal/km-ai/**` → `km-ai-service`（仅内网） |
