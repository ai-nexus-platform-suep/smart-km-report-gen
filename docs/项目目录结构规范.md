## 三、每个微服务标准目录规范

每个微服务子模块统一采用以下 12 层结构：

```
qa-xxx/
├── pom.xml
├── Dockerfile
└── src/main/java/com/qa/xxx/
    ├── XxxApplication.java          # 1. 启动类（该模块唯一 @SpringBootApplication）
    ├── config/                      # 2. @Configuration 配置类
    ├── controller/                  # 3. REST / Feign 内部接口
    ├── service/                     # 4. 业务逻辑接口
    │   └── impl/                    #    （可选）实现类
    ├── client/                      # 5. Feign 接口（声明式，调用其他服务）
    ├── repository/                  # 6. JPA / MyBatis 数据访问层
    ├── entity/                      # 7. 数据库实体（@Entity / @Table）
    ├── dto/
    │   ├── request/                 # 8. 接收请求参数 / Feign 调用入参
    │   └── response/                # 9. Service 返回结果 / Feign 返回值
    ├── vo/                          # 10. 前端视图对象（去敏感字段，格式化展示）
    ├── mapper/                      # 11. MapStruct 转换器（Entity ↔ DTO ↔ VO）
    └── exception/                   # 12. 全局异常处理 + ErrorCode 枚举
```

### 3.1 各层详细说明

| 层 | 作用 | 注意事项 |
|----|------|----------|
| **XxxApplication.java** | Spring Boot 启动入口 | 每个模块只有一个，模块名 + Application |
| **config/** | 各种 @Configuration Bean 注册 | 如 RestClientConfig、ThreadPoolConfig、SecurityConfig |
| **controller/** | 对外 REST 接口 + 内部 Feign 接口 | 不要直接返回 Entity，统一返回 VO 或 ApiResponse |
| **service/** | 核心业务逻辑 | 调用 repository、client，返回 DTO |
| **client/** | Feign 接口声明 | 只声明不实现，由 Spring Cloud 动态代理 |
| **repository/** | 数据库操作 | JPA 用 `extends JpaRepository`，MyBatis 用 `@Mapper` |
| **entity/** | 数据库表映射 | 带 `@Entity @Table @Id @Column`，不出 Controller |
| **dto/request/** | 前端请求体 / Feign 入参 | 带 `@Data` + 校验注解 `@NotNull @NotBlank` |
| **dto/response/** | Service 返回给 Controller / Feign 返回值 | 带 `@Data @Builder` |
| **vo/** | 返回给前端的视图对象 | 裁剪敏感字段（如脱敏 phone、隐藏 passwordHash） |
| **mapper/** | MapStruct 接口 | `@Mapper(componentModel = "spring")`，编译期生成实现 |
| **exception/** | 全局异常处理 | `@RestControllerAdvice` + ErrorCode 枚举 |

### 3.2 数据流转示意

```
前端请求
  │
  ▼
Controller    ← 接收 DTO (Request)，参数校验
  │
  ▼
Service       ← 用 Entity 操作 DB，用 DTO 调 Feign
  │
  ▼
Mapper        ← Entity / DTO 转换为 VO
  │
  ▼
Controller    ← 返回 ApiResponse<VO> 给前端
```

## 四、DTO / VO / Entity / Mapper 分层规范

### 4.1 四层对比

| 层 | 位置 | 职责 | 典型注解 |
|----|------|------|----------|
| **Entity** | `entity/` | 数据库表映射，不暴露到 Controller 之外 | `@Entity @Table @Id @Column` |
| **DTO (request)** | `dto/request/` | 接收前端请求体 / Feign 调用入参 | `@Data` + `@NotNull @NotBlank` |
| **DTO (response)** | `dto/response/` | Service 返回给 Controller 的内部结果 / Feign 返回值 | `@Data @Builder` |
| **VO** | `vo/` | Controller 返回给前端的视图对象，裁剪敏感字段、格式化展示 | `@Data @Builder` |
| **Mapper** | `mapper/` | Entity ↔ DTO ↔ VO 转换，**使用 MapStruct**，编译期生成实现类 | `@Mapper(componentModel = "spring")` |

### 4.2 VO 与 DTO 的核心区别

| 对比 | DTO | VO |
|------|-----|-----|
| 谁用 | Service 层 / Feign 远程调用 | Controller → 前端渲染 |
| 字段 | 完整业务字段（含内部字段） | 裁剪版（去敏感字段 + 格式化） |
| 例子 | `UserInfoResponse(id, username, passwordHash, role)` | `LoginVO(id, username, role)` — 不含 passwordHash |

### 4.3 MapStruct Mapper 示例

```java
@Mapper(componentModel = "spring")
public interface UserMapper {
    UserVO toVO(User entity);
    UserInfoResponse toResponse(User entity);
}
```

## 五、qa-common 公共模块规范

避免各服务重复造轮子，抽取公共代码：

```
qa-common/
└── src/main/java/com/qa/common/
    ├── dto/
    │   ├── ApiResponse.java      # 统一响应包装 { code, message, data }
    │   └── PageResult.java       # 分页响应 { list, total, page, size }
    ├── exception/
    │   ├── BusinessException.java # 业务异常基类
    │   └── ErrorCode.java        # 统一错误码枚举
    ├── util/
    │   └── JsonUtils.java        # Jackson 序列化工具
    └── constant/
        └── SseEventType.java     # SSE 事件类型枚举（thinking/answer/citation/done/error）
```

所有服务 `pom.xml` 引入：

```xml
<dependency>
    <groupId>com.qa</groupId>
    <artifactId>qa-common</artifactId>
    <version>1.0.0</version>
</dependency>
```

## 六、命名规范

### 6.1 包名

| 类型 | 规则 | 示例 |
|------|------|------|
| 基础包 | `com.qa.{模块名}` | `com.qa.core` |
| 子包 | 全小写，单数 | `controller`、`service`、`entity` |

### 6.2 类名

| 类型 | 规则 | 示例 |
|------|------|------|
| 启动类 | 模块名首字母大写 + `Application` | `CoreApplication.java` |
| Controller | 领域名 + `Controller` | `ConversationController.java` |
| Service 接口 | 领域名 + `Service` | `ChatService.java` |
| Service 实现 | 领域名 + `ServiceImpl` | `ChatServiceImpl.java` |
| Repository | 实体名 + `Repository` | `UserRepository.java` |
| Entity | 领域名词单数 | `User.java`, `Message.java` |
| DTO (Request) | 操作名 + `Request` | `LoginRequest.java` |
| DTO (Response) | 操作名 + `Response` | `AuthResponse.java` |
| VO | 领域名 + `VO` | `LoginVO.java`, `DashboardVO.java` |
| Mapper | 实体名 + `Mapper` | `UserMapper.java` |
| Feign Client | 被调用服务名 + `Client` | `SearchServiceClient.java` |
| Config | 功能名 + `Config` | `FeignConfig.java`, `AsyncConfig.java` |
| Exception Handler | 模块名 + `ExceptionHandler` | `CoreExceptionHandler.java` |

### 6.3 方法名

| 操作 | 动词 | 示例 |
|------|------|------|
| 创建 | `create` | `createConversation(CreateConversationRequest req)` |
| 查询单个 | `get` / `find` | `getConversationById(String id)` |
| 查询列表 | `list` | `listConversations(String userId)` |
| 更新 | `update` | `updateConversationTitle(String id, String title)` |
| 删除 | `delete` | `deleteConversation(String id)` |
| 登录 | `login` | `login(LoginRequest req)` |
| 注册 | `register` | `register(RegisterRequest req)` |

## 七、数据库规范

### 7.1 各服务独占表

```
PostgreSQL: qa_db
├── schema: auth        → users
├── schema: core        → conversations, messages
├── schema: search      → document_chunks (只读), retrieval_logs
└── schema: admin       → qa_configs, feedbacks
```

### 7.2 规则

- 每个微服务只拥有自己 schema 下的表，不允许跨服务直接写对方的表
- 需要其他服务的数据，通过 **Feign 接口 / API** 获取，不直接查对方的 DB
- Flyway 迁移脚本统一放在 `src/main/resources/db/migration/`，按 `V1__描述.sql` 命名

### 7.3 字段命名

| 类型 | 规则 | 示例 |
|------|------|------|
| 表名 | 小写下划线，复数 | `users`, `conversations`, `retrieval_logs` |
| 列名 | 小写下划线 | `user_id`, `created_at`, `knowledge_base_id` |
| 主键 | `id` | UUID v4（字符串）或 BIGSERIAL |
| 时间戳 | `created_at`, `updated_at` | TIMESTAMP WITH TIME ZONE |

## 八、资源文件规范

每个服务的 `src/main/resources/` 下：

```
resources/
├── application.yml           # 公共配置（服务名、端口）
├── application-dev.yml       # 开发环境
├── application-prod.yml      # 生产环境
└── db/
    └── migration/
        └── V1__init_xxx.sql  # Flyway 数据库迁移脚本
```

`application.yml` 主配置示例：

```yaml
spring:
  application:
    name: qa-core
  profiles:
    active: dev

server:
  port: 8082
```
