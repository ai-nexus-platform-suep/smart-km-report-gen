# smart-km-report-gen 测试文档

## 1. 概述

本文档描述 smart-km-report-gen（知识管理子系统）的测试策略、测试范围、测试覆盖情况和运行指南。

### 1.1 项目结构

```
smart-km-report-gen/
├── km-common/          # 公共库（常量、DTO、异常、工具类）
│   └── src/test/       # 7 个测试类, 40 个测试用例
├── km-backend/         # 后端服务（Spring Boot + MyBatis）
│   └── src/test/       # 15 个测试类, 132 个测试用例
└── km-frontend/        # 前端（Vue 3，暂无测试）
```

### 1.2 测试框架与依赖

| 模块 | 测试框架 | Mock 框架 | 备注 |
|------|---------|-----------|------|
| km-common | JUnit 5 (Jupiter) | 无 | 纯 POJO/工具类测试 |
| km-backend | JUnit 5 + Mockito | Mockito | MockMvc 用于 Controller 层 |

### 1.3 测试总览

| 测试层级 | 文件数 | 测试用例数 | 状态 |
|---------|--------|-----------|------|
| km-common 单元测试 | 7 | 40 | ✅ 全部通过 |
| km-backend Service 层 | 5 | 60 | ✅ 全部通过 |

## 8. 测试编写规范

### 8.1 命名约定

- 测试类：`{ClassName}Test.java`，放置于与源码相同的包路径下
- 测试方法：`should{ExpectedBehavior}` 格式，使用下划线分隔英文单词

### 8.2 测试结构

每个测试方法遵循 **Given-When-Then** 模式：

```java
@Test
void shouldUpdateKnowledgeBase() {
    // Given — 准备测试数据与 Mock 行为
    KnowledgeBase existing = createKnowledgeBase();
    when(knowledgeBaseMapper.getById("kb-1")).thenReturn(existing);

    // When — 执行被测方法
    KnowledgeBaseVO result = service.update("kb-1", request);

    // Then — 验证结果
    assertNotNull(result);
    verify(knowledgeBaseMapper).updateById(any(KnowledgeBase.class));
}
```

### 8.3 Mock 使用原则

- Service 层：Mock 所有 Mapper/Client 依赖，使用 `@ExtendWith(MockitoExtension.class)`
- Controller 层：Mock 所有 Service 依赖，使用 `MockMvcBuilders.standaloneSetup` 独立模式
- 工具类：不 Mock，直接测试静态方法

### 8.4 测试覆盖要求

- 每个公共方法至少 1 个 Happy Path 测试
- 每个异常路径至少 1 个测试（验证异常类型和错误码）
- 关键边界条件至少 1 个测试（null, 空字符串, 空列表, 最大值, 最小值）
- 涉及外部依赖的方法需覆盖依赖失败时的降级行为
