## Epic：智能问答功能

**描述**：建设基于大语言模型（LLM）和检索增强生成（RAG）技术的智能问答系统，为电力行业用户提供自然语言知识检索与智能对话能力。考核指标：问答准确率。

---

## Feature 1：用户认证（P0）

| 编号 | 用户故事 | 验收标准 |
|------|----------|----------|
| US-1.1 | 作为平台用户，我希望通过用户名和密码注册账号 | 输入用户名密码 → 注册成功；重复用户名提示"已存在" |
| US-1.2 | 作为注册用户，我希望通过用户名和密码登录 | 正确凭据 → 返回 JWT，跳转对话页；错误提示"用户名或密码错误" |
| US-1.3 | 作为登录用户，我希望刷新页面后保持登录状态 | 刷新页面 → Token 有效保持登录，过期跳转登录页 |
| US-1.4 | 作为系统管理员，我希望所有 API 需身份认证，未登录用户无法访问 | 无 Token → 返回 401；前端跳转登录页 |

## Feature 2：SSE 流式对话（P0）

| 编号 | 用户故事 | 验收标准 |
|------|----------|----------|
| US-2.1 | 作为用户，我希望输入问题后 AI 以流式方式逐字返回回答 | Enter 发送 → AI 逐字实时展示，首字延迟 < 2s |
| US-2.2 | 作为用户，我希望能随时停止 AI 正在生成的回答 | 点击停止 → SSE 断开，已生成内容保留 |
| US-2.3 | 作为用户，我希望能对同一条消息重新生成回答 | 点击重新生成 → 替换上一条 AI 回答，重新流式输出 |
| US-2.4 | 作为用户，我希望输入框支持 Enter 发送、Shift+Enter 换行，发送后自动聚焦 | Enter 发送并清空聚焦；Shift+Enter 换行不发送 |
| US-2.5 | 作为用户，我希望新消息到来时页面自动滚到底部 | 新消息到达 → 消息列表自动滚到底 |

## Feature 5：RAG 检索增强生成（P0）

| 编号 | 用户故事 | 验收标准 |
|------|----------|----------|
| US-5.1 | 作为系统，我希望将用户问题向量化后在知识库中检索相关片段 | 输入问题 → 向量化 → pgvector 余弦检索 → 返回 Top K 结果 |
| US-5.2 | 作为系统，我希望将检索结果注入 LLM Prompt，使回答基于知识库内容 | 检索上下文 + System Prompt → LLM 生成引用知识库内容的回答 |
| US-5.3 | 作为用户，我希望看到每个回答的检索来源文档名 | 回答底部展示来源文档名列表 |
| US-5.4 | 作为系统，检索无结果时由 LLM 直接回答并提示未找到相关信息 | 检索为空 → 提示"未找到相关信息"，LLM 直接回答 |
| US-5.5 | 作为系统，检索结果应支持相似度阈值过滤 | 相似度 < 阈值 → 该片段被过滤，不注入上下文 |
